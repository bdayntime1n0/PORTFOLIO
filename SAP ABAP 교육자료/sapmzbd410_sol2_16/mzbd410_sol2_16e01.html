" REPORT PROGRAMME (TYPE 1 PROGRAM)의 INITIALIZATION과 같음.
" 프로그램이 실행될 때 호출되는 이벤트.
" INITIALIZATION을 안쓰는 이유 = REPORT PROGRAMME 이 아니기 때문.

LOAD-OF-PROGRAM.

* GET PARAMETER : SAP MEMORY 에서 메모리 변수의 값을 가져오는 키워드
                     " PARAMETER ID OF AIRLINE CODE.
  GET PARAMETER ID : 'CAR' FIELD SDYN_CONN-CARRID,    "SCREEN 100 - GENERAL의 NAME에서 확인 가능.
                     " PARAMETER ID OF FLIGHT NUMBER.
                     'CON' FIELD SDYN_CONN-CONNID,
                     " PARAMETER ID OF FLIGHT DATE.
                     'DAY' FIELD SDYN_CONN-FLDATE.

" SFLIGHT 에서 데이터를 조회해 WORKAREA( WA_SFLIGHT )에 값을 저장한다.
  SELECT SINGLE *
  FROM ZBD1_FLIGHT16  " SFLIGHT
  INTO WA_SFLIGHT
  WHERE CARRID = SDYN_CONN-CARRID
    AND CONNID = SDYN_CONN-CONNID
    AND FLDATE = SDYN_CONN-FLDATE.

" TRANSACTION CODE 가 100번 화면을 호출하는데,
" 그 때 100번 화면이 출력 전에 PBO 가 먼저 수행되고
" PBO 모듈 중 WA_SFLIGHT 에서 SDYN_CONN 으로 값 복사하는
" MOVE-CORRESPONDING 이 있기 때문에,
" PBD가 끝나고 화면이 출력될 때 SDYN_CONN 에 값이 있어서
" 실행하자마자 100번 화면에 모든 필드에 데이터가 출력되는 것을 볼 수가 있다.
